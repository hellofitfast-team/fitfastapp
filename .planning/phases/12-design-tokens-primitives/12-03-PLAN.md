In `packages/ui/src/page-header.tsx`, make the following changes:

**1. Typography sizing (DS-03 + typography scale):**
Change the h1 from `text-2xl font-bold tracking-tight` to `text-xl font-semibold`:

```tsx
<h1 className="text-xl font-semibold">{title}</h1>
```

Rationale: `text-2xl` (24px) is too large for mobile page titles at 390px. `text-xl` (20px) matches the typography scale documented in Plan 12-01. `font-semibold` (600) replaces `font-bold` (700) for a softer feel. `tracking-tight` removed — not needed at this size and conflicts with Arabic (RTL rules already zero out tracking).

**2. Add subtitle prop (enhancement for Phase 13):**
Add an optional `subtitle` prop alongside `description`:

```tsx
export interface PageHeaderProps {
  title: string;
  subtitle?: string; // NEW — short context line (e.g., "Day 8 of 14")
  description?: string;
  action?: React.ReactNode;
  className?: string;
}
```

Render subtitle between title and description:

```tsx
<h1 className="text-xl font-semibold">{title}</h1>;
{
  subtitle && <p className="text-primary mt-0.5 text-sm font-medium">{subtitle}</p>;
}
{
  description && <p className="text-muted-foreground mt-1 text-sm">{description}</p>;
}
```

The subtitle uses `text-primary` to visually distinguish it from the description (muted). This gives Phase 13 a way to show contextual info like "Day 8 of 14" or "Week 2" without building a custom header.
