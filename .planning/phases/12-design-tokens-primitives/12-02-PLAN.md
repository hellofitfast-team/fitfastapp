  <name>Task 2: Replace reduced-motion block with wildcard approach in client globals.css</name>
  <files>
    apps/client/src/app/globals.css
  </files>
  <action>
In `apps/client/src/app/globals.css`, replace the current reduced-motion block:

**Current (fragile — only covers listed classes):**

```css
/* Reduced Motion */
@media (prefers-reduced-motion: reduce) {
  .animate-fade-in,
  .animate-slide-in,
  .animate-slide-up,
  .animate-slide-down,
  .animate-scale-in {
    animation: none;
  }
}
```

**New (wildcard — covers all animations including stagger delays):**

```css
/* Reduced Motion — covers all animations globally */
@media (prefers-reduced-motion: reduce) {
  *,
  ::before,
  ::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This approach:

- Catches ALL animations (entrance, stagger, future additions) without needing to list each class
- Uses `0.01ms` instead of `0s` to still fire animation events (preventing stuck states)
- Sets `iteration-count: 1` to prevent infinite loops
- Covers transitions and scroll-behavior too
- Is the recommended approach from Tailwind CSS v4 docs
  </action>

  <done>Reduced-motion support is comprehensive — wildcard approach covers all current and future animations.</done>
  </task>
