Create a new file `day-selector.tsx` in the meal-plan `_components` directory. This component will be imported by both meal-plan and workout-plan pages.

**Component API:**

```tsx
interface DaySelectorProps {
  totalDays: number; // 14 for a 14-day plan
  selectedDay: number; // 0-indexed current selection
  onSelectDay: (day: number) => void;
  planStartDate?: string; // ISO date string to compute "today"
  restDays?: number[]; // 0-indexed rest day indices (muted style)
  featureColor?: "nutrition" | "fitness"; // Controls active pill color
}
```

**Implementation:**

- Render a horizontal scrollable strip: `flex gap-1.5 overflow-x-auto pb-1 -mx-4 px-4 scrollbar-hide`
- Each day is a button pill: `flex-shrink-0 min-w-[48px] px-3 py-2.5 rounded-xl text-xs font-semibold transition-colors text-center`
- Active day: `bg-nutrition text-white` (or `bg-fitness text-white` based on featureColor)
- Rest days: `bg-neutral-100 text-neutral-400`
- Normal days: `bg-neutral-100 text-muted-foreground hover:bg-neutral-200`
- Each button shows "Day N" as label (N = day index + 1)
- Below the "Day N" text, show the abbreviated weekday computed from `planStartDate + dayIndex` using `toLocaleDateString(locale, { weekday: 'short' })`. Use `useLocale()` from next-intl for locale-aware weekday names.
- If `planStartDate` is provided, auto-highlight "today" by computing `Math.floor((Date.now() - new Date(planStartDate).getTime()) / 86400000)` clamped to 0-(totalDays-1).
- Auto-scroll the selected day pill into view on mount using `useEffect` + `useRef` on the active button with `scrollIntoView({ inline: 'center', behavior: 'smooth' })`.

Export as named export: `export function DaySelector(...)`.
